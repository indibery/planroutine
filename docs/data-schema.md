# 데이터 스키마

## 입력 데이터: 생산문서등록대장 CSV

나이스(NEIS) 등 학교 행정 시스템에서 내보내기한 CSV.
인코딩 주의: EUC-KR이 기본이므로 UTF-8 변환 필요.

### 컬럼 정의
```
문서번호        | 연현초등학교-17523 형식
결재유형        | 생산, 접수
제목            | 문서 제목 (업무 내용)
기안(접수)자    | 작성자 이름
결재권자        | 최종 결재자
등록일자        | YYYY-MM-DD
수신(발신)자    | 내부결재 / 외부기관명
처리상태        | 종료, 진행 등
게시여부        | Y/N
연계유형        | 일반, 기록물이관, 참고자료 등
전자비전자구분  | 전자
공개구분        | 공개, 부분공개
공개제한근거    | 6호, 7호 등 (부분공개 사유)
첨부파일여부    | Y/N
쪽수            | 숫자
목록공개여부    | 공개
과제명          | 업무 대분류 (일과운영관리, 교육과정계획수립운영 등)
과제카드명      | 업무 소분류
보존기한        | 5년, 10년, 30년, 해외수집
최초발송일자    | YYYY-MM-DD (외부 발송 시)
```

### 앱에서 활용할 핵심 컬럼
- **등록일자** → 시기 패턴 분석, 올해 일정 추천
- **제목** → 업무 내용 표시, 검색
- **과제명/과제카드명** → 카테고리 분류 및 필터
- **결재유형** → 생산(내가 작성) vs 접수(받은 것) 구분
